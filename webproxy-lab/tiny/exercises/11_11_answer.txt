11.11 답

문제 의도

Tiny가 원래 지원하는 메서드는 GET뿐이다.
이 문제에서는 HTTP HEAD 메서드도 지원하게 만들어,
클라이언트가 자원 본문 없이 응답 헤더만 받을 수 있도록 해야 한다.

핵심 아이디어

HEAD 요청은 GET과 거의 동일하게 처리하되,
응답 본문(body)만 보내지 않으면 된다.

즉,
- 상태줄
- 응답 헤더

는 그대로 보내고,
- 파일 내용
- CGI 출력 본문
- 오류 본문 HTML

만 생략하면 된다.

구현 방식

1. doit()에서 GET과 HEAD를 모두 허용한다.

if (strcasecmp(method, "GET") && strcasecmp(method, "HEAD")) {
    ...
}

2. method가 HEAD인지 확인해서 head_only 플래그를 만든다.

head_only = !strcasecmp(method, "HEAD");

3. 이 플래그를 clienterror(), serve_static(), serve_dynamic()에 전달한다.

4. 각 함수에서 head_only가 참이면 본문만 생략한다.

구체적인 동작

1. serve_static()
- 기존처럼 Content-length, Content-type 등 응답 헤더는 보낸다.
- 하지만 head_only이면 파일 내용을 읽어서 쓰지 않고 바로 return 한다.

2. clienterror()
- 기존처럼 에러 헤더는 보낸다.
- 하지만 head_only이면 HTML body는 전송하지 않는다.

3. serve_dynamic()
- HEAD 요청에서는 CGI 본문을 실행해서 보내지 않는다.
- 최소한의 응답 헤더만 보내고 return 한다.

검증 방법

1. 빌드

cd /workspaces/jungle-sw-ai-webproxy-lab-docker/webproxy-lab/tiny
gcc -I . -g -Wall -o tiny_11_11 exercises/tiny_11_11.c csapp.o -lpthread

2. 실행

./tiny_11_11 8000

3. telnet 또는 raw TCP로 HEAD 요청

HEAD /home.html HTTP/1.0

4. 기대 결과

응답에는 다음과 같은 헤더만 오고,
HTML 본문은 오지 않아야 한다.

HTTP/1.0 200 OK
Server: Tiny Web Server
Connection: close
Content-length: 127
Content-type: text/html

정리

11.11의 핵심은
GET 처리 흐름을 거의 그대로 재사용하면서,
HEAD 요청일 때는 본문만 생략하도록 Tiny를 확장하는 것이다.
